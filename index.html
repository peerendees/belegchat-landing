Die Technologie dahinter

Threema Work Gateway

Das Herzstück von BelegChat. Jeder Kunde erhält eine dedizierte Threema-Gateway-ID — einen verschlüsselten Eingangskanal, der ausschließlich für die beiden Buchungskreise (Mandanten) dieses Kundenkontos existiert. Nachrichten und Anhänge werden Ende-zu-Ende-verschlüsselt übertragen. Kein Server liest mit, bevor die Entschlüsselung auf der BelegChat-Seite stattfindet. Damit setzt BelegChat bewusst auf maximale Vertraulichkeit.

Threema Work

BelegChat nutzt Threema Work als Kommunikationsbasis. Damit läuft auch die Kundenkommunikation — Onboarding, Support, Updates — ausschließlich über denselben verschlüsselten Kanal. Kein E-Mail-Ticketsystem, keine externen Plattformen.

Häufige Fragen

  • BelegChat ist eine digitale Buchhaltungshilfe für Selbstständige und Kleinunternehmer in Deutschland. Du schickst deine Belege per Threema ein — BelegChat erkennt sie automatisch, ordnet sie einem Buchungskonto zu und erstellt eine DATEV-Exportdatei für deinen Steuerberater. Alles GoBD-konform, DSGVO-sicher, ohne Papierchaos.
  • Kundenkonto: Dein BelegChat-Zugang. Ein Kundenkonto entspricht einem Vertrag und kann bis zu zwei Mandanten umfassen.
    Mandant / Buchungskreis: Ein Mandant ist ein buchhalterisch getrennter Bereich — z. B. eine Firma oder ein Gewerbe. Jeder Mandant hat seine eigene Threema-Anbindung und seine eigenen Belege sowie Buchungssätze. Zwei Mandanten unter einem Konto sind strikt getrennt: Kein gegenseitiger Einblick, keine gemeinsamen Daten.
  • Threema ist der einzige Messenger, der Ende-zu-Ende-Verschlüsselung mit vollständiger Anonymität verbindet — keine Telefonnummer, keine E-Mail-Adresse erforderlich. Für Belege mit sensiblen Geschäftsdaten ist das keine Komfort-Entscheidung, sondern eine datenschutzrechtliche. BelegChat nutzt darüber hinaus Threema Work Gateway: Jedes Kundenkonto erhält eine eigene, dedizierte Eingangs-ID. Was du einschickst, bleibt ausschließlich in deinen Buchungskreisen (Mandanten) dieses Kontos.
  • Threema Work Gateway ist eine API-Schnittstelle, über die Unternehmen verschlüsselte Nachrichten senden und empfangen können — programmatisch, automatisiert, ohne dass ein Mensch mitlesen kann. BelegChat nutzt diese Schnittstelle, um für jedes Kundenkonto einen eigenen, verschlüsselten Eingangskanal zu betreiben und setzt damit bewusst auf maximale Vertraulichkeit.
  • Zu Beginn des Onboardings erstellst du ein Threema-Backup, das auf dem Server gespeichert wird. Dieses Backup ist fester Bestandteil der Einrichtung und unumgänglich. Deine Threema-ID geht nur dann verloren, wenn das Kennwort für dieses Backup verloren geht. So ist dein Zugang zum BelegChat-Eingang bestmöglich abgesichert.
  • Ein Kundenkonto unterstützt bis zu zwei Mandanten — zum Beispiel ein Unternehmen und ein zweites Geschäft, oder zwei Partner mit je einer eigenen Threema-ID. Jeder Buchungskreis ist strikt getrennt: Kein gegenseitiger Einblick, keine gemeinsamen Daten.
  • BelegChat stellt die technischen Mittel bereit, die eine GoBD-konforme Arbeitsweise ermöglichen: Belege werden unveränderlich archiviert, alle Änderungen werden protokolliert, verbuchte Buchungssätze sind dauerhaft gesperrt. Die Verantwortung für die Einhaltung der GoBD liegt beim Nutzer — BelegChat ist Werkzeug und Auftragsverarbeiter, keine Buchhaltungssoftware.
  • Nein. Dein Steuerberater erhält eine Standard-DATEV-Exportdatei, die er direkt in DATEV Unternehmen online importieren kann. Kein neues Programm, keine Schulung, kein zusätzlicher Aufwand auf seiner Seite.
  • BelegChat befindet sich in der aktiven Entwicklungsphase. Beta-Tester nutzen das Produkt zu einem dauerhaft vergünstigten Preis und geben aktiv Rückmeldung. Im Gegenzug erhalten sie priorisierten Support direkt über Threema Work und dauerhaften Zugang zum vergünstigten Tarif — auch nach dem offiziellen Launch.
  • Personenbezogene Daten und Buchungsdaten werden in getrennten Datenbanken gespeichert (Zwei-Datenbank-Architektur mit Pseudonymisierung). Ein Kundenkonto unterstützt bis zu zwei Mandanten. Belegdateien liegen verschlüsselt im Storage-System. Alle eingehenden Daten kommen Ende-zu-Ende-verschlüsselt über Threema — kein Klartext-Eingang über unsichere Kanäle. Für alle externen Dienstleister (Supabase, Stripe u. a.) bestehen Auftragsverarbeitungsverträge gemäß Art. 28 DSGVO.
  • BelegChat kommt ohne separates Upload-Portal aus: Deine Belege schickst du direkt per Threema — kein separates Hochlade-Formular, kein zusätzlicher Kanal. Die Oberfläche, in der du Belege prüfst, Buchungskonten zuweist und den DATEV-Export erstellst, ist eine Progressive Web App (PWA): Du nutzt BelegChat im Browser oder installierst die PWA optional auf deinem Gerät („Zum Startbildschirm hinzufügen“). Kein App-Store, keine native Installation. Der Eingang für Belege bleibt dabei immer Threema — der Messenger, dem du ab nun vertraust.
